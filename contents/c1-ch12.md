# Chapter 12: Single Page Apps

## Section 1: What Is a Single Page App?

Up to this point, most of the websites you’ve built have worked like traditional
web pages: the browser loads a page, you click a link, and a new page loads.
This process repeats every time you move to a different part of the site.

A **Single Page App (SPA)** works differently.

In a single page app, the browser loads **one main page**, and JavaScript
controls everything that happens after that. Instead of loading new pages, the
app updates parts of the existing page.

The user experiences smooth transitions and instant updates, even though they
never leave the original page.


### Traditional Websites vs Single Page Apps

Some sites are better in the traditional method of separate HTML pages
served from a central server, and honestly they're probably easier to build.
However, some user experiences can be achieved via SPAs that aren't possible
with the traditional static sites.

Let’s compare the two approaches.

**Traditional Website:**

1. User clicks a link
2. Browser requests a new page
3. Server sends new HTML
4. Page reloads
5. User sees new content

**Single Page App:**

1. User clicks a button
2. JavaScript runs
3. Page content changes
4. No reload occurs

From the user’s perspective, SPAs can feel faster and more responsive.

### Real-World Examples

Many popular websites use the single page app model, including:

* Gmail
* Google Docs
* Trello
* Discord
* Spotify Web Player

When you use these sites, you rarely see full page reloads. Most changes happen
instantly through JavaScript.

### Why Use Single Page Apps?

Single page apps are popular because they offer several advantages:

* Speed: No repeated page loading
* Smooth experience: Fewer interruptions
* Better interactivity: Instant feedback
* Modern design: App-like behavior in the browser

These features make SPAs ideal for tools, dashboards, games, and productivity
apps.

> **Helpful Hint:**
> A single page app is not about having only one screen. It’s about managing
> multiple “screens” inside one HTML file using JavaScript.

### Tradeoffs and Challenges

SPAs also come with challenges:

* More JavaScript code to manage
* Harder to debug if poorly organized
* Requires careful planning of state and structure
* Can be confusing if code becomes messy

This is why good design and organization matter more in SPAs than in simple
websites.
