## Section 5: Basic App Architecture

As single page apps grow, they can quickly become difficult to manage if the
code is not organized well. **App architecture** refers to how you structure
your files, functions, and data so that your program stays readable, reliable,
and easy to improve.

Good architecture does not mean complicated code. It means clear structure and
consistent organization**.

### Separating Responsibilities

A well-designed app separates different kinds of work:

* Data - information your app stores (state)
* Logic - how your app processes data
* Presentation - what the user sees (HTML and CSS)

Even in small projects, keeping these ideas separate makes your code easier to
understand.

Example:

```js
let appState = {
  notes: [],
  currentView: "home"
};

function addNote(text) {
  appState.notes.push(text);
}

function renderNotes() {
  // update the page using appState
}
```

Here:

* `appState` stores data
* `addNote()` changes data
* `renderNotes()` updates the display

### Organizing Your Functions

Group related functions together.

For example:

* Navigation functions
* Data functions
* Display functions
* Event handlers

This makes it easier to find and fix problems later.

> **Helpful Hint:**
> If you have trouble naming a function, it probably does too many things. Split
> it into smaller parts.

### Avoiding Repeated Code

Repeated code leads to bugs and wasted time. Instead, use functions to handle
common tasks.

Bad example:

```js
homeView.style.display = "none";
notesView.style.display = "none";
settingsView.style.display = "none";
```

Better example:

```js
function hideAll() {
  for (let v of document.querySelectorAll(".view")) {
    v.style.display = "none";
  }
}
```

### Naming Conventions

Clear names improve readability.

Good examples:

* `loadNotes()`
* `saveSettings()`
* `renderHome()`
* `handleSubmit()`

Avoid vague names like:

* `doStuff()`
* `thing()`
* `x()`

### Keeping Files Organized (Preview)

In larger projects, you may split code into files:

```
index.html
style.css
app.js
storage.js
ui.js
```

For our purposes, we'll keep working in one large file.


### Planning Before Coding

Before writing code, sketch your design:

* What views exist?
* What data is stored?
* Which functions control behavior?
* How do parts connect?

A few minutes of planning can save hours of debugging.
