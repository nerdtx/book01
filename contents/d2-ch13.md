## Section 2: Build the HTML as an App Shell

A single page app needs a stable page structure before any logic is added. This
stable structure is often called the **app shell**. The shell gives the
application a predictable layout and gives JavaScript clear places to update.

For this Battleship app, the shell must support a two-board game. That means the
page needs more than one display area.

Your app shell should include:

* a title
* a message area
* a scoreboard
* a player board
* a computer board
* a control area

These parts will stay on the page while JavaScript updates their contents during play.

### The Core HTML Structure

Start with this HTML:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Simple Battleship</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <main
      <h1>Simple Battleship</h1>

      <section
        <p
      </section>

      <section
        <p>Player Hits: <span
        <p>Player Misses: <span
        <p>Computer Hits: <span
        <p>Computer Misses: <span
      </section>

      <section
        <div class="boardArea">
          <h2>Your Board</h2>
          <div
        </div>

        <div class="boardArea">
          <h2>Computer Board</h2>
          <div
        </div>
      </section>

      <section
        <button
      </section>
    </main>

    <script src="script.js"></script>
  </body>
</html>
```

This gives the app all of the major regions it will need.

### Why Two Board Containers Matter

This is not just a styling choice. The game has two separate views of state:

* the **player board** shows where the computer has fired and where the player’s ships are
* the **computer board** shows where the player has fired

These boards serve different purposes, so they should be separate in the HTML as well.

That separation will make your rendering code much cleaner later.

### The Message Area

The `<p

It can display messages like:

* "Click a square on the computer board to fire."
* "Hit!"
* "Miss!"
* "Computer scored a hit!"
* "You win!"
* "The computer wins!"

A dedicated message area is a simple but important part of a good single page
app. It helps users understand what is happening.

### The Scoreboard

The scoreboard gives the player immediate feedback about the current state of the game.

In this version, it tracks:

* player hits
* player misses
* computer hits
* computer misses

These values will be updated by JavaScript as the game progresses.

### The Board Regions

Each board lives inside a `.boardArea` container with a heading. This has two benefits:

* it makes the page easier to read
* it gives CSS a clean structure to style later

The board containers themselves, `#playerBoard` and `#computerBoard`, start out
empty. JavaScript will fill them with clickable grid cells.

### The Control Area

The New Game button lives in its own section. This keeps controls separate from
the play area and makes the app easier to expand later.

For example, you might eventually add:

* a Reset Saved Game button
* a difficulty selector
* a ship placement mode

Because the controls are grouped together, those additions will fit naturally.

> **Helpful Hint:**
> In a single page app, your HTML should provide stable “slots” for JavaScript
> to update. If you give each major part of the app a clear home, your code
> becomes easier to manage.

### HTML First, Behavior Later

At this stage, the app still does nothing interactive. That is fine. Right now,
your goal is to create a clean shell that supports the game.

A strong app shell makes the rest of the project easier because JavaScript will always know:

* where to display messages
* where to show scores
* where to draw each board
* where to connect controls

In the next section, you will style this structure so the two boards appear as
proper game grids and the app starts to look like a real Battleship interface.
