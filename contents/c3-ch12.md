## Section 3: Hiding, Showing, and Saving Page Data

In a single page app, you don’t load new pages when the user moves around.
Instead, you **hide and show parts of the page** and update stored data as
needed. This section teaches you how to control what users see and how to save
information between visits.

### Showing and Hiding Elements

One of the simplest ways to control visibility is by changing an element’s
`display` style.

Example HTML:

```html
<div id="homeView">Home Screen</div>
<div id="notesView">Notes Screen</div>
```

Example JavaScript:

```js
function showHome() {
  document.getElementById("homeView").style.display = "block";
  document.getElementById("notesView").style.display = "none";
}

function showNotes() {
  document.getElementById("homeView").style.display = "none";
  document.getElementById("notesView").style.display = "block";
}
```

Only one section is visible at a time, creating the illusion of multiple
“pages.”

> **Helpful Hint:**
> Start by hiding all views, then show only the one you want. This prevents
> overlapping content.

### Using CSS Classes to Control Visibility

Another clean approach is to use CSS classes.

CSS:

```css
.hidden {
  display: none;
}
```

JavaScript:

```js
const homeView = document.getElementById("homeView");
homeView.classList.add("hidden");
const notesView = document.getElementById("notesView");
notesView.classList.remove("hidden");
```

This keeps style rules in CSS and logic in JavaScript.

### Introducing `localStorage`

So far, all of your app’s data disappears when the page reloads. To fix this,
browsers provide localStorage, a built-in way to save small amounts of
data.

`localStorage` stores data as text and keeps it even after the browser
closes.

### Saving Data to `localStorage`

Use `setItem()` to save data:

```js
localStorage.setItem("username", "JoeySchmoey");
```

Retrieve it with `getItem()`:

```js
let name = localStorage.getItem("username");
console.log(name); // "JoeySchmoey"
```

### Saving Complex Data with JSON

Because `localStorage` only stores strings, arrays and objects must be
converted using JSON.

Saving:

```js
let notes = ["Buy milk", "Study JavaScript"];

localStorage.setItem("notes", JSON.stringify(notes));
```

Loading:

```js
let savedNotes = JSON.parse(localStorage.getItem("notes"));

if (savedNotes !== null) {
  notes = savedNotes;
}
```

> **Helpful Hint:**
> Always check for `null` when loading from `localStorage`. The item may not
> exist yet.

---

### Clearing Stored Data

To remove a single item:

```js
localStorage.removeItem("username");
```

To clear everything:

```js
localStorage.clear();
```

Use this carefully, especially in larger apps.

### Why This Matters

By combining:

* Show/hide logic
* App state
* Local storage

You can now build apps that:

* Feel like real software
* Remember user data
* Work across browser sessions

This is a major step toward professional-quality web applications.
