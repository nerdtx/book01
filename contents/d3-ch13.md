## Section 3: Style the App Shell and Boards

Now that the HTML shell is in place, the next step is to make the app look like
a game instead of a plain document. In a single page app, CSS is responsible for
the layout and appearance, while JavaScript is responsible for behavior. Keeping
those jobs separate is part of good structure.

This section will style:

* the overall app layout
* the scoreboard
* the two board areas
* the cells inside each board
* the visual states for hits and misses

### Style the Overall App

Start with some basic layout rules:

```css
body {
  font-family: Arial, sans-serif;
  max-width: 900px;
  margin: 0 auto;
  padding: 20px;
}

#app {
  display: grid;
  gap: 20px;
}

h1, h2 {
  margin: 0;
}

#statusPanel,
#scorePanel,
#controls {
  padding: 10px;
  border: 1px solid #cccccc;
  background-color: #f8f8f8;
}
```

This gives the app a centered layout with clear spacing between regions.

### Style the Scoreboard

Since the scoreboard contains several related values, a horizontal flex layout works well:

```css
#scorePanel {
  display: flex;
  gap: 20px;
  flex-wrap: wrap;
}
```

This keeps the score values readable and allows them to wrap if the screen becomes narrower.

### Lay Out the Two Boards

Now place the player board and computer board side by side:

```css
#boardsPanel {
  display: grid;
  grid-template-columns: 1fr 1fr;
  gap: 30px;
}

.boardArea {
  display: grid;
  gap: 10px;
}
```

This gives each board its own space while keeping them aligned.

> **Helpful Hint:**
> A two-column layout is a natural choice here because the player needs to
> compare both boards during the game.

### Create the Board Grids

Each board needs to be a 6 by 6 grid of square cells:

```css
#playerBoard,
#computerBoard {
  display: grid;
  grid-template-columns: repeat(6, 50px);
  grid-template-rows: repeat(6, 50px);
  gap: 5px;
}
```

The `repeat(6, 50px)` pattern creates six equal columns and six equal rows.

### Style the Cells

Each square on the board will be represented by a `<div>` with the class `cell`.

```css
.cell {
  display: flex;
  align-items: center;
  justify-content: center;
  border: 1px solid #333333;
  background-color: #dfefff;
  font-weight: bold;
  cursor: pointer;
  user-select: none;
}
```

This makes the cells square, centered, and clickable.

### Visual States for Hits, Misses, and Ships

Your JavaScript will later add CSS classes to indicate different states.

```css
.cell.hit {
  background-color: #ff9999;
}

.cell.miss {
  background-color: #cccccc;
}

.cell.ship {
  background-color: #99ccff;
}
```

These classes have different purposes:

* `.hit` marks a successful shot
* `.miss` marks an unsuccessful shot
* `.ship` will be used on the **player board only** so the player can see their own ships

The computer’s ships should remain hidden during play.

### Style the Button

Give the New Game button a little structure:

```css
button {
  padding: 10px 16px;
  font-size: 16px;
  cursor: pointer;
}
```

This keeps controls readable and easy to click.

### Why This CSS Structure Works

Notice that the CSS is organized around page regions and reusable classes:

* `#app`, `#scorePanel`, `#boardsPanel` define layout
* `.boardArea` groups related board content
* `.cell`, `.hit`, `.miss`, `.ship` define reusable cell styles

This is good single page app design because the CSS is not guessing what the app
will do. It is simply defining how known interface pieces should look.

Later, JavaScript will update classes and content, and the CSS will take care of
the visual result.
