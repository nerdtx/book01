## Section 4: Working with Navigation

In a traditional website, navigation usually means clicking links that load new
pages. In a single page app, navigation works differently. Instead of loading
new pages, JavaScript changes which parts of the page are visible. This creates
the feeling of moving between screens without leaving the page.

### Simulating Navigation with Buttons

The simplest form of navigation uses buttons or links that trigger JavaScript
functions.

Example HTML:

```html
<nav>
  <button id="homeBtn">Home</button>
  <button id="notesBtn">Notes</button>
  <button id="settingsBtn">Settings</button>
</nav>
```

Example JavaScript:

```js
document.getElementById("homeBtn").addEventListener("click", showHome);
document.getElementById("notesBtn").addEventListener("click", showNotes);
document.getElementById("settingsBtn").addEventListener("click", showSettings);
```

Each button activates a function that displays the correct view.

### Centralizing Navigation Logic

Instead of having many separate functions, you can centralize navigation in one
place.

Example:

```js
function navigate(view) {
  hideAllViews();

  document.getElementById(view).style.display = "block";
}

function hideAllViews() {
  document.querySelectorAll(".view").forEach(div => {
    div.style.display = "none";
  });
}
```

Now you can navigate like this:

```js
navigate("homeView");
navigate("notesView");
```

> **Helpful Hint:**
> Centralizing navigation makes your app easier to update and debug as it
> grows.

### Optional: Hash-Based Navigation

Browsers allow you to change the URL hash without reloading the page.

Example URL:

```text
example.com/#notes
```

You can read (and set) this in JavaScript:

```js
let currentView = window.location.hash;
window.location.hash = "newview";
```

And respond to changes:

```js
window.addEventListener("hashchange", function() {
  navigate(location.hash.substring(1) + "View");
});
```

This allows users to bookmark or refresh while staying on the same “page.”

### Visual Feedback for Navigation

Good apps show users where they are.

Example:

```js
function setActive(button) {
  document.querySelectorAll("nav button").forEach(btn => {
    btn.classList.remove("active");
  });

  button.classList.add("active");
}
```

You can use CSS to highlight the active button.

### Why Navigation Design Matters

Clear navigation:

* Prevents user confusion
* Makes apps feel professional
* Reduces mistakes
* Improves usability

A well-designed SPA should always make it obvious where the user is and how to
move around.
