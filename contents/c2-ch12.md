## Section 2: The Idea of “State” in Your App

As your applications become more interactive, they need to **remember things**.
They need to know what the user has done, what choices were made, and what
should be shown next. All of this information is called the **state** of the
app.

In a single page app, managing state is one of your most important
components of the application.

### What Is State?

**State** is all the data that describes what your app looks like and how it
behaves at a given moment.

Examples of state include:

* Which screen is currently visible
* What text the user has typed
* Which items are in a list
* Whether the user is logged in
* Which button was last clicked

If any of these change, the state changes.

### A Simple Example

Imagine a to-do app with this information:

```js
let todos = ["Finish homework", "Practice guitar"];
let currentView = "list";
```

Here:

* `todos` stores the tasks
* `currentView` tracks what screen is active

Together, these variables represent the app’s state.

If the user adds a task:

```js
todos.push("Read chapter 12");
```

The state changes.

### Why State Matters in SPAs

In traditional websites, the server usually controls state. Each page load
resets things.

In SPAs, the browser controls state. Nothing reloads automatically, so
JavaScript must track everything.

If you lose track of state:

* Buttons stop working
* Screens show wrong data
* User input disappears
* Bugs appear

Well-managed state leads to reliable apps.

> **Helpful Hint:**
> If you ever feel confused about what your app is doing, write down the current
> values of your main variables. That’s your state.

### Centralizing Your State

As apps grow, it’s best to keep important state in one place.

Example:

```js
let appState = {
  view: "home",
  notes: [],
  user: null
};
```

This makes it easier to understand and update your app’s behavior.

### Updating State and the Interface

A key idea in SPAs is this:

> When state changes, the page should update.

Example:

```js
function addNote(text) {
  appState.notes.push(text);
  renderNotes();
}
```

Here:

1. State changes
2. Display updates

Keeping this pattern makes your apps easier to maintain.

### Planning Before Coding

Before writing a SPA, ask:

* What data do I need to remember?
* Where will I store it?
* When will it change?
* What parts of the page depend on it?

Answering these questions first saves a lot of time later.
