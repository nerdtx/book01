## Section 5: Reading Values from Form Inputs

JavaScript’s ability to read values from form elements allows your web page to
react to what a user types, selects, or clicks. This section teaches how to
collect input from text fields, dropdown menus, radio buttons, and checkboxes,
and how to respond when the user submits a form.

### Reading from Text Inputs

Let's jump right into it.

Most forms include text inputs where users can type information. To access what
a user types, use the `.value` property on the DOM element.

**Example:**

```html
<input id="nameInput" type="text" placeholder="Enter your name">
<button onclick="greetUser()">Greet</button>

<script>
  function greetUser() {
    const name = document.getElementById("nameInput").value;
    alert("Hello, " + name + "!");
  }
</script>
```

> **Helpful Hint:**
> Always access `.value` on the input element to retrieve what the user typed.
> Other elements don't have a .value so don't be surprised if you try to pull
> data from a non-form element with .value and it's blank.

### Reading from Dropdowns

Dropdown menus are represented with `<select>` elements. You still use `.value`
to retrieve the selected option.

**Example:**

```html
<select id="animalChoice">
  <option value="dog">Dog</option>
  <option value="cat">Cat</option>
  <option value="llama">Llama</option>
</select>
```

```js
const selectedAnimal = document.getElementById("animalChoice").value;
```

### Radio Buttons

Radio buttons are grouped by giving them the same `name` attribute. The
recommended method is to use `querySelector()` to find which one is checked.

**Example:**

```html
<input type="radio" name="color" value="red"> Red<br>
<input type="radio" name="color" value="blue"> Blue<br>
```

```js
const selectedColor = document.querySelector('input[name="color"]:checked').value;
```

> **Helpful Hint:**
> Only one radio button in a group can be selected at a time, which makes it
> perfect for choosing one option from a set.  Keep in mind, a LOT of options
> can make the radio unwieldy and you may want a select instead.

### Checkboxes

Checkboxes are different from radio buttons - users can select multiple
checkboxes at once. You can use `.checked` to test whether a checkbox is
selected.

**Example:**

```html
<input type="checkbox" id="subscribeBox"> Subscribe to newsletter
```

```js
const isSubscribed = document.getElementById("subscribeBox").checked;
```

### Handling Form Submission

Rather than relying on a button’s `onclick`, you can respond to the `<form>`
being submitted. This is useful when handling multiple inputs together.

**Example:**

```html
<form id="infoForm">
  <input type="text" id="username" placeholder="Username">
  <button type="submit">Submit</button>
</form>

<script>
  const form = document.getElementById("infoForm");

  form.addEventListener("submit", function(event) {
    event.preventDefault(); // Prevents the page from reloading
    const user = document.getElementById("username").value;
    console.log("Submitted user:", user);
  });
</script>
```

### Basic Validation

Before submitting or processing the data, it’s common to check that a value was
actually entered.

The following example is laughably simple and examines the data previously
pulled from the form.

**Example:**

```js
if (user === "") {
  alert("Please enter a username.");
}
```

> **Helpful Hint:**
> Form validation is essential for both user experience and security. Start
> simple now - you’ll see more powerful methods later.
