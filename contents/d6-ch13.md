## Section 6: Build Utility Functions for Ship Placement

Now that the app has a stable shell, a centralized state object, and separate
render functions, it is time to build one of the most important parts of the
game: **ship placement**.

In Battleship, ships cannot just appear anywhere. They must follow rules:

* ships stay inside the board (duh.)
* ships are placed either horizontally or vertically
* ships do not overlap (crash!)

This is a good place to reinforce a key single page app principle:

**Utility functions should solve small, focused problems without touching the DOM.**

Ship placement is game logic. It should not update the page directly. It should
only calculate valid data and return it.

### Step 1: Create a Random Number Helper

A simple helper function makes the rest of the code easier to read.

```js
function randomInt(max) {
  return Math.floor(Math.random() * max);
}
```

This returns a whole number from `0` up to, but not including, `max`.

Examples:

* `randomInt(6)` might return `0`, `1`, `2`, `3`, `4`, or `5`
* `randomInt(2)` might return `0` or `1`

You will use this repeatedly when choosing random rows, columns, and ship orientations.

### Step 2: Decide the Ship Sizes

This version of Battleship uses three ships:

* one ship of length 3
* two ships of length 2

Store those ship lengths in an array:

```js
const shipLengths = [3, 2, 2];
```

This makes it easy to loop through and place each ship.

### Step 3: Represent a Ship Clearly

Each ship should be stored as an object with a `cells` array.

Example:

```js
let ship = {
  cells: [
    { row: 2, col: 1 },
    { row: 2, col: 2 },
    { row: 2, col: 3 }
  ]
};
```

This structure is simple and works well with the rest of the app. It also makes
hit detection easier later.

### Step 4: Generate Candidate Cells

Before you can place a ship, you need a way to generate the cells it would occupy.

A ship placement depends on:

* a starting row
* a starting column
* a direction
* a length

This function builds the list of cells for one possible ship:

```js
function buildShipCells(startRow, startCol, length, horizontal) {
  let cells = [];

  for (let i = 0; i < length; i++) {
    cells.push({
      row: horizontal ? startRow : startRow + i,
      col: horizontal ? startCol + i : startCol
    });
  }

  return cells;
}
```

If `horizontal` is `true`, the ship extends across columns. If `horizontal` is `false`, it extends down rows.

### Step 5: Check Whether a Placement Is Valid

Not every randomly generated placement will work. Some ships may run off the
board, and some may overlap existing ships.

To prevent that, create a validation function.

```js
function isValidPlacement(size, cells, occupied) {
  for (let cell of cells) {
    const key = `${cell.row},${cell.col}`;

    if (
      cell.row < 0 ||
      cell.row >= size ||
      cell.col < 0 ||
      cell.col >= size ||
      occupied.has(key)
    ) {
      return false;
    }
  }

  return true;
}
```

This function checks two things:

1. every cell is inside the board
2. no cell overlaps a location that is already occupied

The `occupied` value should be a `Set` containing strings like `"2,3"` to
represent already-used board positions.

> **Helpful Hint:**
> A `Set` is useful here because it makes overlap checks fast and easy. If a
> coordinate string is already in the set, that square is already taken. A `Set`
> is like an array that doesn't allow repeated values.

### Step 6: Place One Ship at a Time

Now create a function that keeps trying random positions until it finds one
valid placement for a single ship length.

```js
function placeSingleShip(size, length, occupied) {
  while (true) {
    const horizontal = Math.random() < 0.5;
    const startRow = randomInt(size);
    const startCol = randomInt(size);

    const cells = buildShipCells(startRow, startCol, length, horizontal);

    if (isValidPlacement(size, cells, occupied)) {
      for (let cell of cells) {
        occupied.add(`${cell.row},${cell.col}`);
      }

      return { cells: cells };
    }
  }
}
```

This function works like this:

* choose a random direction
* choose a random starting point
* build the candidate ship
* test whether it fits
* if it works, mark those spaces as occupied and return the ship
* if it does not work, try again

The `while (true)` loop is acceptable here because the function always exits
when it finds a valid placement.

### Step 7: Place a Full Fleet

Now use the helper functions to build a complete fleet.

```js
function placeFleet(size) {
  let ships = [];
  let occupied = new Set();

  for (let length of shipLengths) {
    let ship = placeSingleShip(size, length, occupied);
    ships.push(ship);
  }

  return ships;
}
```

This returns an array of ship objects.

Example result:

```js
[
  {
    cells: [
      { row: 0, col: 1 },
      { row: 0, col: 2 },
      { row: 0, col: 3 }
    ]
  },
  {
    cells: [
      { row: 3, col: 4 },
      { row: 4, col: 4 }
    ]
  },
  {
    cells: [
      { row: 5, col: 0 },
      { row: 5, col: 1 }
    ]
  }
]
```

### Step 8: Use the Fleet Generator for Both Sides

Because both the player and the computer need ships, the same function can be reused twice.

Later, during game setup, you will use:

```js
gameState.playerShips = placeFleet(gameState.size);
gameState.computerShips = placeFleet(gameState.size);
```

This is a good example of **reusability**. One utility function handles the same
job for both sides.

### Why This Section Matters for SPA Structure

Notice what these functions do **not** do:

* they do not change the HTML
* they do not update the scoreboard
* they do not show messages
* they do not handle button clicks

They only build valid ship data.

That is good structure.

A single page app becomes easier to maintain when utility functions stay focused
on data and rules, while rendering and events are handled elsewhere.
