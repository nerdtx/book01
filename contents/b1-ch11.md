# Chapter 10: JavaScript & HTML Interactivity

## Section 1: The DOM - What JavaScript Sees

When you write an HTML page, you're creating a structured document. The browser
takes that document and builds something called the **DOM**, or **Document
Object Model**. You can think of the DOM as a big tree full of things.

Each of those *things* is an element, with each element (or object) (a `<div>`,
a `<p>`, a `<button>`, etc.) acting like a fork in the branch of that tree, or
the leaf on the end of a branch.  JavaScript uses this tree to understand what’s
on the page - and more importantly, to change it.

### What Is the DOM?

The DOM is a live model of the current state of the HTML document. Every time
the browser loads a web page, it builds a representation of the structure and
content of that page in memory.

If your HTML looks like this:

```html
<html>
<head>
<title> my page </title>
</head>
<body>
  <div>
    <h1>Hello!</h1>
    <p>This is a paragraph.</p>
  </div>
</body>
</html>
```

The browser builds a DOM that looks like this:

![DOM Example](imgs/ch11-dom.png "DOM Example"){width=300px height=285px}


Each tag becomes an **element node**. Each of those nodes can be seen, read, or
changed by JavaScript.

> **Helpful Hint:**
> The DOM is not your HTML file. It’s a copy of it that the browser keeps in
> memory and updates as needed. JavaScript doesn’t change your file - it changes
> the DOM.

### Why Should You Care About the DOM?

Because the DOM is what JavaScript sees and interacts with, every bit of
interactivity you want to add - whether it's hiding a button, showing a popup,
changing a color, or loading new content - must go through the DOM.

In this chapter we will discuss (among other things), how to:

* Select elements from the DOM
* Read and change their content
* Respond to user actions like clicks or typing

All of this relies on first understanding that the DOM is the gateway between
your code and the page the user sees.
